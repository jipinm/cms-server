1.前言
本文档为统一身份认证平台作为奇瑞集团项目权威的主数据中心及统一的认证入口，便于其他应用系统与统一身份认证平台进行用户同步的规范阐述。

2.术语解释
统一身份认证系统是专注于身份管理与访问控制的系统，主要提供覆盖奇瑞集团的组织架构服务、人员信息服务及访问应用系统的认证服务。

5.如何接入
应用系统集成IAM共分为两部分：身份认证以及数据同步集成，其中应用数据同步采用连接器的方式完成。连接器由统一身份认证平台提供，厂商需实现接口规范章节中指定的接口。
第三方应用系统接入统一身份认证平台后，根据统一身份认证定义的标准JSON数据格式接口规范与统一身份认证平台做数据交互。接口规范定义了机构、账号等对象增、删、改、查的抽象方法，第三方应用系统根据集成范围做对应的业务实现。
下文将以账号和组织机构对象做示例。
统一身份认证平台与第三方集成应用系统同步，数据交互如下图：

统一身份平台通过内置连接器请求第三方集成应用系统的API接口服务，请求调用接口定义的接口方法。
下游集成应用在UserCreateService、UserUpdateService、UserDeleteService方法体内实现对本应用系统账号数据的增、改、删操作。统一身份认证平台调用对应接口在下游应用中进行应用账号的增、删、改操作。
下游集成应用在QueryAllUserIdsService、QueryUserByIdService方法体内实现对本应用系统账号数据的查询操作。统一身份认证平台调用对应接口在下游应用中进行应用账号的查询操作。不需要回收下游系统账号的应用无需实现此方法。
下游集成应用在OrgCreateService、OrgUpdateService、OrgDeleteService方法体内实现对本应用系统组织机构数据的增、改、删操作。统一身份认证平台调用对应接口在下游应用中进行组织机构的增、删、改操作。下游集成应用可以只实现必要的接口，不需要的接口可以不实现。
下游集成应用在QueryAllOrgIdsService、QueryOrgByIdService方法体内实现对本应用系统组织机构数据的查询操作。统一身份认证平台调用对应接口在下游应用中进行应用组织机构的查询操作。不需要回收下游系统组织机构的应用无需实现此方法。

6.2.安全性说明
6.2.1.接口协议
应用系统提供HTTP（S）协议的账号管理接口。
6.2.2.用户认证
在调用接口时，为了防止非法调用，需要在每次请求中添加授权信息（如账号名、密码）。第三方应用服务端为统一身份认证平台系统提供接口调用的授权账号和密码。
统一身份认证平台每次调用接口时，提供授权账号名、密码，报文字段详见SchemaService接口的报文字段说明。应用系统需要保证提供的授权账号密码的复杂度较高，建议为大写字母、小写字母、数字、特殊字符4选4的非弱口令。
第三方应用服务端收到请求，校验授权账号和密码，校验通过则执行请求，校验不通过则返回错误提示。
6.2.3.报文加密
在调用接口时，为了防止数据泄露，可以对请求报文进行加密（密钥为16位）。统一身份认证平台每次调用接口时，对发送的报文内容进行加密。第三方应用服务端收到请求，对报文内容解密，并将响应报文加密后返回给统一身份认证平台。报文格式为JSON，默认不加密，对称加密类型可选范围有AES、AESNET和SM4。
6.2.4.签名验证
在调用接口时，为了防止数据篡改，可以在每次请求中添加签名信息。默认无签名，参数签名类型可选范围有MD5和SM3。
若设置了签名类型，则统一身份认证平台每次调用接口时，对发送的报文内容进行签名，并把签名信息加入报文中，报文字段详见SchemaService接口的报文字段说明。
第三方应用服务端收到请求，对签名进行校验，校验不通过则返回错误提示。
签名的实现请参见附件10.1中的banboocloud_Codec-0.0.2.jar。
6.3.SchemaService接口
6.3.1.功能说明
SchemaService的功能是定义第三方目标系统中“账号”等对象全部属性信息，包括属性名称、类型、是否必填字段、是否多值。用以建立起统一身份认证平台用户字段和三方目标系统账号字段的映射关系。集成的三方目标系统必须要实现此接口。
6.3.2.请求报文说明
6.3.2.1报文示例

{
	"bimRequestId": "9e928d12ec8a4c1bb75283b8df71308d",
	"bimRemotePwd": "********",
	"bimRemoteUser": "********",
	"signature": "8dfb0b7f7fccb823a7896e24a9fb8008"
}

6.3.2.2报文字段说明
bimRequestId ：BIM每次调用生成的随机ID，应用系统每次响应返回此ID，String类型。
bimRemoteUser ：BIM调用三方应用接口的授权账号，由应用分配给BIM系统，String类型。
bimRemotePwd ：BIM调用三方应用接口的密码，由应用分配给BIM系统，String类型。
signature：BIM每次调用生成的签名，应用系统需对签名进行验证，String类型。
注：报文编码格式统一采用UTF-8。

6.3.3.响应报文说明
6.3.3.1报文示例
{
	"bimRequestId": "9e928d12ec8a4c1bb75283b8df71308d",
	"account": [{
		"multivalued": false,
		"name": "attr1",
		"required": true,
		"type": "String"
	}, {
		"multivalued": false,
		"name": "attr2",
		"required": true,
		"type": "String"
	}, {
		"multivalued": false,
		"name": "attr3",
		"required": true,
		"type": "String"
	},……],
	"organization": [{
		"multivalued": false,
		"name": "attr4",
		"required": true,
		"type": "String"
	}, {
		"multivalued": false,
		"name": "attr5",
		"required": true,
		"type": "String"
	}, {
		"multivalued": false,
		"name": "attr6",
		"required": true,
		"type": "String"
	}]
}

6.3.3.2报文字段说明
bimRequestId :BIM每次调用接口发送的请求ID，字段为String类型。
account/organization/role :此接口定义的对象类型，可选值为account（账号）、organization（组织机构）、role（角色），如果只做账号管理，organization等其它对象可以不用定义。字段为数组类型。
name ：定义对象的属性字段名称。字段为String类型。
type ：定义对象的属性字段类型，可选值为String、int、double、float、long、byte、boolean。字段为String类型。
required : 定义对象的属性字段在创建时是否为必填字段。可选值true或者false。字段为boolean类型。
multivalued ：定义对象的属性字段是否为多值。可选值true或者false。字段为boolean类型。
注：报文编码格式统一采用UTF-8。
7.用户管理接口
7.1.UserCreateService接口
7.1.1.功能说明
UserCreateService接口是应用系统的账号创建方法。统一身份管理系统调用应用系统实现的该接口用于在应用系统中创建指定账号。
7.1.2.请求报文说明
7.1.2.1报文示例
{
	"bimRequestId": "11928d12ec8a4c1bb75283b8df71308d",
	"bimRemoteUser": "********",
	"bimRemotePwd": "********",
	" attr1": "zhangsan",
	" attr2": "000001",
	" attr3": "张三",
	"signature": "8dfb0b7f7fccb823a7896e24a9fb8008"
}

7.1.2.2报文字段说明
bimRequestId/bimRemotePwd/bimRemoteUser :BIM接口调用的必传字段，说明见SchemaService。
attr1/attr2/ attr3：三方应用系统通过SchemaService接口中定义的账号字段属性。
signature：BIM每次调用生成的签名，应用系统需对签名进行验证，String类型。
注：报文编码格式统一采用UTF-8。
7.1.3.响应报文说明
7.1.3.1报文示例
成功：
{
	"bimRequestId": "11928d12ec8a4c1bb75283b8df71308d",
	"uid": "89746776",
	"resultCode": "0",
	"message": "success"
}
失败：
{
	"bimRequestId": "11928d12ec8a4c1bb75283b8df71308d",
	"resultCode": "500",
	"message": "账号创建失败，报错信息自定义"
}
7.1.3.2报文字段说明
uid ：作为用户成功推送到下游系统，下游返回的一个成功标识，可以将统一用户推送过去的id或者username赋值给uid，返回给统一用户。
resultCode ：接口调用处理的结果码，0为正常处理，其它值由应用系统定义。字段为String类型，必传字段。
message ：接口调用处理的信息。字段为String类型。
bimRequestId :BIM每次调用接口发送的请求ID，字段为String类型，必传字段。
注：报文编码格式统一采用UTF-8。
7.2.UserUpdateService接口
7.2.1.功能说明
UserUpdateService接口是应用系统的账号修改方法。统一身份管理系统调用应用系统实现的该接口用于在应用系统中更新指定的账号。
7.2.2.请求报文说明
7.2.2.1基本信息修改报文示例
{
	"bimRequestId": "22928d12ec8a4c1bb75283b8df71308d",
	"bimRemoteUser": "********",
	"bimRemotePwd": "********",
	"bimUid": "89746776",
	" attr1": "lisi",
	" attr2": "李四",
	"signature": "8dfb0b7f7fccb823a7896e24a9fb8008"
}
7.2.2.2启用、禁用报文示例
{
	"bimRequestId": "22928d12ec8a4c1bb75283b8df71308d",
	"bimRemoteUser": "********",
	"bimRemotePwd": "********",
	"bimUid": "89746776",
	" __ENABLE__": false,
	"signature": "8dfb0b7f7fccb823a7896e24a9fb8008"
}
7.2.2.3报文字段说明
bimRequestId/bimRemotePwd/bimRemoteUser :BIM接口调用的必传字段，说明见SchemaService。
bimUid: 三方应用系统账号创建时，返回给BIM应用系统的账号主键uid。必传字段。
attr1/ attr2：需要修改的账号字段属性，默认只会传递修改过的属性。
signature：BIM每次调用生成的签名，应用系统需对签名进行验证，String类型。
__ENABLE__ ：账号状态属性, __ENABLE__：true启用,__ENABLE__：false 禁用。
注：报文编码格式统一采用UTF-8。
7.2.3.响应报文说明
7.2.3.1报文示例
成功：
{
	"bimRequestId": "22928d12ec8a4c1bb75283b8df71308d",
	"resultCode": "0",
	"message": "success"
}
失败：
{
	"bimRequestId": "11928d12ec8a4c1bb75283b8df71308d",
	"resultCode": "500",
	"message": "账号更新失败，报错信息自定义"
}
7.2.3.2报文字段说明
resultCode ：接口调用处理的结果码，0为正常处理，其它值由应用系统定义。字段为String类型，必传字段。
message ：接口调用处理的信息。字段为String类型。
bimRequestId :BIM每次调用接口发送的请求ID，字段为String类型，必传字段。
注：报文编码格式统一采用UTF-8。
